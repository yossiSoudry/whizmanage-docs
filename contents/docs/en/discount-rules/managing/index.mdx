## Managing and Organizing Discount Rules

Efficient management of discount rules is crucial for running successful promotions. In this guide, we'll learn how to leverage all the tools the system offers for professional and efficient management.

### Table Interface

<Step>
<StepItem title="Interface Overview">

The discount rules table displays consolidated information about all rules:

**Main columns:**
- **Name** - Rule name
- **Type** - Discount type + settings button
- **Status** - Enable/disable toggle
- **Start/End date** - Active dates
- **Priority** - Priority order
- **Filters** - Product filtering + edit button

- **Message** - Edit message
- **Start/End date** - Change dates

**Toggles:**
- **Status** - Instant enable/disable of rule

**Edit buttons:**
- **Type Settings** (‚öôÔ∏è) - Open discount settings
- **Filters Edit** - Edit product filters

### Bulk Edit

<Step>

1. Check the checkbox next to each rule
2. Or click the checkbox in the header to select all

**Number of selected rules** appears in the bar at the bottom of the screen.

<img src="https://placehold.co/800x400?text=Select+rows" alt="Select rows" />
</StepItem>

<StepItem title="Actions on Multiple Rules">

When rules are selected, a **Floating Toolbar** appears at the bottom with options:

**Available actions:**
- üìã **Duplicate** - Duplicate selected rules
- ‚öôÔ∏è **Bulk Edit** - Bulk editing
- üóëÔ∏è **Move to Trash** - Move to trash
- ‚ôªÔ∏è **Restore** (in trash) - Restore rules
- üî¥ **Delete Permanently** - Permanent deletion

<img src="https://placehold.co/800x400?text=Bulk+actions" alt="Bulk actions" />

<StepItem title="Using Bulk Edit">

**Bulk Edit** allows editing multiple rules simultaneously:

1. Select desired rules
2. Click **Bulk Edit** from Toolbar
3. Choose fields to edit:
   - **Status** - Change status for all
   - **Priority** - Increase/decrease priority
   - **Start/End date** - Update dates
   - **Message** - Replace message
4. Set new values
5. Click **Apply Changes**

<img src="https://placehold.co/800x400?text=Bulk+edit+modal" alt="Bulk edit modal" />

**Useful scenarios:**
- Extend promotion - Update End date for all rules
- Temporary disable - Change Status to Draft
- Update priorities - Increase Priority by 10

### Duplicating Rules

<Step>
<StepItem title="Duplicate Single Rule">

Duplicating a rule is the fastest way to create a similar rule:

1. Select one rule
2. Click **Duplicate** in Toolbar
3. Rule will be duplicated with "(copy)" added to name
4. Edit the duplicated rule as needed

<img src="https://placehold.co/800x400?text=Duplicate+rule" alt="Duplicate rule" />

**What's duplicated:**
- All settings and parameters
- Filters
- Conditions
- Dates (same as original)

**What changes:**
- Status changes to **Draft**
- "(copy)" added to name

<Note title="Tip" type="info">
After duplicating, it's recommended to change dates and name immediately to avoid confusion.
</Note>
</StepItem>

<StepItem title="Multiple Duplication">

Duplicating multiple rules at once (Pro):

1. Select multiple rules
2. Click **Duplicate**
3. All rules will be duplicated with "(copy)"
4. Edit using Bulk Edit

**Useful for:**
- Creating series of similar promotions
- Copying rules to new cycle
- Creating variations of same promotion

<img src="https://placehold.co/800x400?text=Duplicate+multiple" alt="Duplicate multiple" />
</StepItem>
</Step>

### Trash Management

<Step>
<StepItem title="Move to Trash">

Instead of permanent deletion, rules move to trash:

**Move to trash:**
1. Select rule(s)
2. Click **Move to Trash**
3. Confirm action

**What happens:**
- Rule hidden from main table
- Rule saved in trash for 30 days
- Can restore rule anytime

<img src="https://placehold.co/800x400?text=Move+to+trash" alt="Move to trash" />
</StepItem>

<StepItem title="View Trash">

**Trash** button at top of table:

**In trash you can:**
- See all deleted rules
- Restore rules (Restore)
- Delete permanently (Delete Permanently)

<img src="https://placehold.co/800x400?text=View+trash" alt="View trash" />

<Note title="Important" type="warning">
Rules in trash are automatically deleted after 30 days. Permanent deletion is irreversible!
</Note>
</StepItem>

<StepItem title="Restore Rules">

Restore rule from trash:

1. Enter trash (Trash button)
2. Select rule(s) to restore
3. Click **Restore**
4. Rule returns to main table

**What's restored:**
- All original settings
- Status returns to original state
- Dates remain as they were

<img src="https://placehold.co/800x400?text=Restore+from+trash" alt="Restore from trash" />
</StepItem>
</Step>

### Organization by Priority

<Step>
<StepItem title="Understanding Priority">

**Priority** determines rule execution order:

**Rules:**
- **Lower** number = **Higher** priority
- Priority 0 runs before Priority 10
- Rules with same Priority run by ID

**Example:**
| Priority | Rule Name | Execution Order |
|----------|-----------|-----------------|
| 0 | VIP 30% Discount | 1 (first) |
| 5 | Regular 15% Discount | 2 |
| 10 | Basic 5% Discount | 3 (last) |

<img src="https://placehold.co/800x400?text=Priority+order" alt="Priority order" />
</StepItem>

<StepItem title="Strategic Organization">

**Organization recommendations:**

**VIP Layer (Priority 0-9):**
- VIP customer discounts
- Special promotions
- Loyalty rules

**General Layer (Priority 10-19):**
- Regular promotions
- Category discounts
- BOGO and bundles

**Base Layer (Priority 20+):**
- Default discounts
- Basic free shipping
- General rules

<img src="https://placehold.co/800x400?text=Priority+strategy" alt="Priority strategy" />

<Note title="Tip" type="info">
Leave gaps between Priority values (0, 10, 20) to have room to insert additional rules in future.
</Note>
</StepItem>
</Step>

### Troubleshooting Common Issues

<Step>
<StepItem title="Rule Not Working">

**Check:**
1. ‚úÖ Status is Active (not Draft)
2. ‚úÖ Dates are correct (within current range)
3. ‚úÖ Filters contain correct products
4. ‚úÖ Conditions are met in cart
5. ‚úÖ Priority not blocked by another rule

<img src="https://placehold.co/800x400?text=Rule+not+working" alt="Rule not working" />
</StepItem>

<StepItem title="Two Rules Conflicting">

**Solution:**
1. Check Priority - rule with lower Priority runs first
2. Change Filters so they don't overlap
3. Adjust Conditions to prevent conflict
4. Consider merging into one rule

<img src="https://placehold.co/800x400?text=Rules+conflict" alt="Rules conflict" />
</StepItem>

<StepItem title="Changes Not Saving">

**Possible causes:**
- Communication issue - check internet connection
- Permission error - verify editing permissions
- Session expired - log in again

**Solution:**
1. Refresh page
2. Log in again
3. Try editing again

<img src="https://placehold.co/800x400?text=Changes+not+saving" alt="Changes not saving" />
</StepItem>
</Step>
