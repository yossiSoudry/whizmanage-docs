## Importing Products from WhizManage Files

The WhizManage system enables easy and efficient transfer of products between different systems using dedicated export files. This option is ideal for synchronization between multiple stores, data backup, or transferring products between development and production environments.

### Preparing a WhizManage Export File

<Step>
<StepItem title="Exporting Products from Source System">

1. In the source system, navigate to the product management screen
2. Select the products you want to export or choose "All Products"
3. Click on the "Export" button (download icon)
4. Select "WhizManage File" from the menu
5. Save the created file to your computer

<img src="https://placehold.co/800x300?text=Export+from+source+system" alt="Export from source system" />

<Note title="Tip" type="info">
You can use filters and search options to export only relevant products.
</Note>
</StepItem>

<StepItem title="Required Preparations for Import">

Before importing, verify:
1. That product types are compatible between systems
2. That user permissions allow product importing
3. That there is enough storage space for products and images

<img src="https://placehold.co/800x300?text=Preparations+for+import" alt="Preparations for import" />

<Note title="Important" type="warning">
Check that the target system version is compatible with or newer than the source system version for maximum compatibility.
</Note>
</StepItem>
</Step>

### Import Process in the Target System

<Step>
<StepItem title="Starting the Import">

1. In the target system, navigate to the product management screen
2. Click on the "Import" button (upload icon)
3. Select "WhizManage File" from the menu

<img src="https://placehold.co/800x300?text=Start+import+in+target+system" alt="Starting import in target system" />
</StepItem>

<StepItem title="Choosing Import Type">

Choose the appropriate import type:
1. **New Products** - to create new products in the system
2. **Update Products** - to update existing products by ID or SKU

<img src="https://placehold.co/800x300?text=Choose+import+type" alt="Choosing import type" />

<Note title="Important" type="warning">
Selecting "New Products" will create new copies even if similar products already exist in the system.
</Note>
</StepItem>

<StepItem title="Uploading the File">

1. Drag the WhizManage file to the designated area or click to select a file
2. The system will check the file's validity
3. You'll receive a confirmation message if the file was accepted successfully

<img src="https://placehold.co/800x300?text=Upload+WhizManage+file" alt="Uploading WhizManage file" />
</StepItem>

<StepItem title="Starting File Processing">

After successfully uploading the file:
1. Click on the "Start Import" button
2. The system will begin processing the file
3. A progress indicator will be displayed during processing

<img src="https://placehold.co/800x300?text=Start+file+processing" alt="Starting file processing" />

<Note title="Tip" type="info">
The import process may take time depending on the number of products and images in the file.
</Note>
</StepItem>
</Step>

### What Does a WhizManage File Contain?

A dedicated WhizManage file contains all the information required for complete product transfer, including:

1. **Basic Product Information**
   - Name, price, description
   - SKU, inventory, categories

2. **Advanced Information**
   - Product attributes and options
   - Variations and relationships between them
   - Tags and SEO settings

3. **Images and Media**
   - Links to product images
   - Product galleries and variation images
   - Downloadable files (if any)

4. **Relationships Between Products**
   - Related products
   - Upsell products
   - Grouped products

<Note title="Important" type="warning">
A WhizManage file is a unique format specific to the WhizManage system and cannot be manually edited. Any changes may damage the file.
</Note>

### Advanced Tips for Importing from WhizManage Files

1. **Creating Backups**
   - Back up the target system before significant imports
   - Keep copies of import files for documentation purposes
   - Document the import process and decisions made

2. **Importing in Stages**
   - For large systems, consider importing in small groups
   - First perform a test import with a few products
   - Arrange products by categories or types

3. **Post-Import Tests**
   - Check a representative sample of imported products
   - Verify that relationships between products are maintained
   - Check that images and media were uploaded properly

<Note title="Tip" type="info">
When transferring an entire system to a new instance, start by importing system settings and templates before importing products.
</Note>
